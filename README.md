# 🪨 Eternal Stone

**Eternal Stone** é um pequeno jogo experimental feito em **HTML, CSS e JavaScript puro**.

O jogador controla algo incomum:

> **Uma pedra.**

Ela não anda.
Ela não pula.
Ela não luta.

Ela **apenas existe… e resiste ao tempo.**

O jogo é uma analogia sobre **resiliência, adaptação e pressão do ambiente**.

# 🎮 Conceito do Jogo

No mundo real muitas vezes não controlamos:

* o ambiente
* as circunstâncias
* o tempo

Assim como uma pedra na natureza, o jogador precisa **se adaptar às condições ao redor**.

Durante o jogo a pedra enfrentará ambientes como:

* ❄️ Montanha fria
* 🌊 Rio
* 💧 Cachoeira
* 🌊 Fundo do mar
* 🧱 Subsolo profundo

Cada ambiente exige uma forma diferente de adaptação.

---

# ⏳ Mecânica do Jogo

O tempo passa constantemente.

```
1 dia = 4 segundos
```

A cada dia a pedra sofre desgaste natural.

O jogador precisa escolher **como reagir ao ambiente**.

Sistema de integridade:

| Evento             | Efeito |
| ------------------ | ------ |
| Passagem de um dia | -5 HP  |
| Escolha correta    | +4 HP  |
| Escolha incorreta  | -10 HP |

O jogo termina quando:

* 💥 **HP chega a 0** → a pedra quebra
* 🏁 **30 dias são completados**

---

# 💎 Final Secreto

Se o jogador terminar o jogo com:

```
HP > 100
```

Depois de suportar pressão, desgaste e tempo…

A pedra se transforma em:

**💎 DIAMANTE**

---

# 🎓 Tutorial Inicial

O jogo começa com instruções clássicas de jogos:

```
Use WASD ou setas para mover
Use espaço para pular
```

Mas ao tentar…

Nada acontece.

Porque **pedras não se movem**.

Esse momento reforça a ideia central do jogo:

> Nem sempre controlamos o ambiente.
> Mas podemos decidir **como resistir a ele**.
